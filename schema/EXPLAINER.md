## What this app does

An interactive React diagram that visualises the financial and operational structure of a **Dual-Rail Humanitarian Cat Bond** — a catastrophe bond that settles on two parallel tracks (traditional fiat and blockchain tokens) and routes payouts automatically to first-mile beneficiaries via mobile money when a parametric trigger fires.

The app has three tabs:

| Tab | Content |
|---|---|
| **Structure** | Clickable, draggable SVG node-and-edge diagram. Click any stakeholder node to see its role in a side panel. Toggle between Global South view (recipients at top) and Global North view (finance at top). |
| **Rails** | Side-by-side comparison table of the fiat rail vs. the token rail (settlement, minimum investment, KYC, secondary market). |
| **Trigger** | Numbered step-by-step sequence of what happens from the moment a parametric event is detected to the moment beneficiaries receive cash. |

## Key concepts

| Term | Definition |
|---|---|
| **SPV** | Special Purpose Vehicle — the single legal entity that issues both fiat notes and security tokens against the same collateral pool |
| **Parametric trigger** | A rule-based payout condition (e.g. earthquake magnitude ≥ 6.5) evaluated against an objective index, with no loss adjustment needed |
| **Collateral pool** | The bond principal, held in liquid low-risk assets (T-bills, tokenized MMFs) for the duration of the bond term |
| **Calculation agent** | Licensed entity that formally certifies whether the trigger index has been breached; their attestation is binding for both rails |
| **Oracle** | On-chain data feed (e.g. Chainlink) that ingests official event data (USGS, NOAA) and makes it available to the smart contract |
| **Fiat rail** | Traditional settlement track: ISIN/CUSIP notes, DTCC settlement, wire coupons, institutional investors ($100k–$250k min) |
| **Token rail** | Blockchain settlement track: ERC-3643 security tokens, USDC coupons, compliant ATS trading, lower minimum ($100–$1k) |
| **ERC-3643** | The T-REX protocol — a security token standard with KYC/AML rules embedded directly in the token transfer logic |
| **Humanitarian wallet** | Pre-authorized wallet held by a humanitarian org (IFRC, WFP) that receives the payout automatically on trigger |
| **MNO / DFS** | Mobile Network Operator / Digital Financial Services — the first-mile infrastructure for cash-out to beneficiaries |

## Section-by-section walkthrough of `catbond-diagram.jsx`

### 1. COLORS

```js
const COLORS = { bg, surface, border, gold, green, blue, red, ... }
```

A single flat object holding all hex color values. Every style in the component references `COLORS.*` — no hardcoded hex strings elsewhere. This ensures a color change in one place propagates everywhere.

Semantic aliases:
- `fiat` = `gold` — used on all fiat rail UI elements
- `crypto` = `purpleBright` — used on all token rail elements
- `trigger` = `redBright` — reserved for trigger-event highlights

### 2. nodeInfo

```js
const nodeInfo = { sponsors, spv, collateral, oracle, calcagent,
                   fiat_investors, token_investors, humanitarian,
                   mno, person, mno_agent, super_agent, bank, central_bank }
```

A lookup table of stakeholder metadata. Each key is the node `id` used in click handlers. Fields:

| Field | Purpose |
|---|---|
| `title` | Display name in SVG box and info panel |
| `sub` | Sub-label (smaller text, institutions or standard names) |
| `color` | Accent color for the node's left bar, border, and info panel |

| `special` | `true` only on `spv` — renders with a tinted green fill to mark it as the central entity |
| `rail` | `"fiat"` or `"crypto"` — marks which settlement track the investor node belongs to |

### 5. CatBondDiagram — component state

```js
const [selected,    setSelected]   = useState("spv");       // active node id
const [activeView,  setActiveView] = useState("structure"); // active tab
const [pos,         setPos]        = useState(SOUTH_POS);   // current node positions
const [view,        setView]       = useState("south");      // "north" | "south"
const posRef   = useRef(SOUTH_POS);  // stable read for animation start-point
const dragRef  = useRef(null);       // drag state (avoids re-renders during drag)
const animRef  = useRef(null);       // requestAnimationFrame handle
```

- `pos`: the canonical position of every node. Updated by both drag events and the animation loop.
- `posRef`: a ref that mirrors `pos` — the animation function reads from `posRef.current` for its start snapshot rather than a stale closure over `pos`.
- `view`: tracks which perspective is active, used to label the toggle button correctly.

---

### 6. updatePos helper

```js
function updatePos(next) {
  setPos(next);
  posRef.current = next;
}
```

Always call `updatePos` instead of `setPos` directly. This keeps `posRef` in sync so that if an animation starts immediately after a drag, it reads the correct start position.

---

### 7. animateTo — view-flip animation

```js
const animateTo = useCallback((targetPos) => {
  if (animRef.current) cancelAnimationFrame(animRef.current);
  const startPos  = posRef.current;          // snapshot at animation start
  const startTime = performance.now();
  const DURATION  = 700;                     // ms

  function step(now) {
    const raw = Math.min((now - startTime) / DURATION, 1);
    // ease-in-out cubic: t = raw<0.5 ? 4raw³ : 1-(−2raw+2)³/2
    const t = raw < 0.5 ? 4*raw**3 : 1 - (-2*raw + 2)**3 / 2;
    const next = {};
    for (const id in targetPos) {
      next[id] = {
        x: startPos[id].x + (targetPos[id].x - startPos[id].x) * t,
        y: startPos[id].y + (targetPos[id].y - startPos[id].y) * t,
      };
    }
    updatePos(next);
    if (raw < 1) animRef.current = requestAnimationFrame(step);
  }
  animRef.current = requestAnimationFrame(step);
}, []);
```

Because both nodes and edges derive from `pos`, updating `pos` inside `requestAnimationFrame` causes both to animate together — no separate edge animation is needed.

### 8. handleToggle — view switcher

```js
function handleToggle() {
  const next = view === "south" ? "north" : "south";
  setView(next);
  animateTo(next === "north" ? NORTH_POS : SOUTH_POS);
}
```

One function call: flip the `view` label, then kick off the 700ms animation.

### 9. Toggle button

```jsx
<button className={`cbd-toggle cbd-toggle--${view}`} onClick={handleToggle}>
  <span className={`cbd-toggle__icon ${view === "north" ? "rotated" : ""}`}>⇅</span>
  {view === "south" ? "Global North View" : "Global South View"}
</button>
```

CSS classes `cbd-toggle--south` and `cbd-toggle--north` switch the gradient and glow animation:

| State | Gradient | Glow animation |
|---|---|---|
| South (default) | Rose → violet | `glow-south` — rose/crimson pulse |
| North | Teal → indigo → violet | `glow-north` — teal/blue pulse |

The `⇅` icon rotates 180° when in North view via the `rotated` class.

### 10. Draggable nodes

```js
function handleMouseDown(e, id) {
  e.preventDefault();
  const svg = svgRef.current;
  dragRef.current = { id, startMouse: svgPoint(svg, e) };

  function onMove(ev) {
    const pt = svgPoint(svg, ev);
    const { id, startMouse } = dragRef.current;
    updatePos(prev => ({
      ...prev,
      [id]: { x: prev[id].x + (pt.x - startMouse.x),
               y: prev[id].y + (pt.y - startMouse.y) }
    }));
    dragRef.current.startMouse = pt;
  }
  // ... addEventListener / removeEventListener on window
}

function svgPoint(svg, e) {
  const pt = svg.createSVGPoint();
  pt.x = e.clientX; pt.y = e.clientY;
  return pt.matrixTransform(svg.getScreenCTM().inverse());
}
```

`svgPoint` converts screen coordinates to SVG viewBox coordinates, correctly accounting for any CSS scaling of the SVG element.

### 12. Dynamic zone bounds

```js
function nodeBounds(ids, pos, pad = 14) {
  const xs = ids.flatMap(id => [pos[id].x, pos[id].x + NW]);
  const ys = ids.flatMap(id => [pos[id].y, pos[id].y + NH]);
  return { x: Math.min(...xs)-pad, y: Math.min(...ys)-pad,
           w: Math.max(...xs)-Math.min(...xs)+2*pad,
           h: Math.max(...ys)-Math.min(...ys)+2*pad };
}

const FINANCIAL_IDS = ["collateral","oracle","sponsors","spv","calcagent",
                        "fiat_investors","token_investors","humanitarian"];
const PAYOUT_IDS    = ["mno","person","mno_agent","super_agent","bank","central_bank"];
```

Zone background rectangles are rendered as `<rect>` elements whose `x/y/width/height` are recomputed from `pos` on every render. This means zones resize and move in real time during both drag and animation.

### 15. Structure tab — SVG diagram

The SVG uses `viewBox="0 0 960 1040"` with `width="100%"` so it scales responsively to the full window width.

**`<defs>` block:**
- `#grid` pattern: a subtle dot-grid background drawn with a `<path>` in `<pattern>`
- Arrow markers (`arr-gold`, `arr-green`, `arr-blue`, `arr-red`, `arr-fiat`, `arr-crypto`): one per color, referenced by `markerEnd` on each `<path>`

**Zone rectangles:**
Two dashed `<rect>` + `<text>` pairs whose bounds are computed dynamically from `pos`:
- `FINANCIAL STRUCTURE` (teal zone) — wraps SPV, collateral, oracle, investors, sponsors, humanitarian
- `PAYOUT CHAIN` (rose zone) — wraps MNO, Recipient, MNO Agent, Super Agent, Bank, Central Bank

**Edges:**
Each edge is a `<path>` (cubic bezier) + a `<text>` label. Faces are specified per edge to control exit/entry direction.

| Edge | Meaning |
|---|---|
| Sponsors → SPV | Risk premium payment |
| SPV → Collateral | Principal deposited |
| Collateral → Oracle | Event monitoring (dashed) |
| Oracle → Calc Agent | Raw data feed |
| Calc Agent → SPV | Signed trigger attestation |
| SPV → Fiat Tranche | Notes issued to institutional investors |
| SPV → Token Tranche | ERC-3643 tokens minted |
| SPV → Humanitarian Wallet | Conditional payout on trigger (dashed) |
| Humanitarian → MNO | Disbursement |
| MNO → Recipient | Mobile payout |
| Recipient → MNO Agent | Cash-out |
| MNO → MNO Agent | Agent network (dashed) |

**Nodes:**
Each node is a `<g>` (SVG group) at `translate(pos[id].x, pos[id].y)`. Contents:
1. Background `<rect>` — border color switches between `COLORS.border` (default) and the node's accent color (selected)
2. Left-edge accent bar — a narrow `BAR`px `<rect>` in the node color
3. Title `<text>` in `FONT_SERIF`
4. Sub-label `<text>` in `FONT_MONO`

The SPV node additionally renders a glow ring `<rect>` when selected, and always has a tinted green background fill.

### 16. Structure tab — info panel

A 220px fixed-width column to the right of the SVG. Contains:

1. **Selected node card** — only rendered when `selectedNode` is truthy. Shows `title`, `sub`, and `desc` from `nodeInfo[selected]`. Border is tinted to the node's color at 25% opacity (`${color}40` in hex).
2. **Legend** — maps each `COLORS.*` entry to its semantic label, rendered as colored 12×3px bars.
3. **Hint** — "Click any node to explore…"

---

### 17. Rails tab

Two cards in a CSS grid (`1fr 1fr`). Each card is built from an array of `[label, value]` pairs. Each row is a flex container with the label left-aligned (uppercase, `textDim`) and the value right-aligned (`text`), separated by a bottom border.

---

### 18. Trigger tab

A vertical list of 7 numbered steps rendered from an array of `{ n, color, title, desc }` objects. Each step card has:
- A large faded number (opacity 0.4) as a visual anchor
- A colored title in `FONT_SERIF`
- A description in `COLORS.textDim`

Colors follow the responsible actor at each step (blue = oracle/data, gold = calc agent, red = payout, fiat/crypto = investor loss absorption).

---

### 19. Footer

A single `<div>` with a top border and a legal disclaimer string. No state dependency.
